# 트루셰 타일링 (Truchet Tiling)

> **[AI Implementation Context]**
> 이 문서는 해당 알고리즘을 다른 프로그래밍 언어(C++, Python, Unity, Rust 등)나 그래픽스 환경으로 이식(Porting)할 때 참조하기 위한 기하학/알고리즘 명세서입니다. 특정 언어의 그래픽스 API에 종속되지 않은 순수 논리와 수학적 규칙을 담고 있습니다.


## 1. 개요 및 목적 (Overview)
**한 종류의 정사각 타일**을 무작위 방향(2가지)으로 격자에 배치하면 미로 같은 무늬가 창발한다. 이 스케치는 3가지 타일 변형(사분원 호 / 대각선 / 삼각형)을 3개 띠로 보여준다.

## 2. 핵심 이론 및 원리 (Core Concept & Math)
극도로 단순한 규칙 — "정사각형 한 칸에, 두 방향 중 하나를 무작위로 그린다" — 하나에서 무한한 복잡성이 나온다. 각 칸의 상태는 **불리언 하나**(방향 A/B)면 충분하다.

세 변형에서 "방향"의 의미:
- **arcs(호)** : 서로 마주보는 두 모서리를 잇는 사분원 두 개. 두 배치 중 하나 → 곡선 미로.
- **diagonals(대각선)** : `\` 또는 `/` 대각선 하나.
- **triangles(삼각형)** : 정사각형을 대각선으로 나눈 두 삼각형 중 하나를 검게.

## 3. 알고리즘 의사코드 (Pseudocode)
```
for each cell (i, j):
    flip = random_bool()      // 이 칸의 방향
    x = i*S,  y = j*S
    if style == arcs:
        if flip: 좌상↔우하 호 두 개
        else:    우상↔좌하 호 두 개
    if style == diagonals:
        line( flip ? (좌상→우하) : (우상→좌하) )
    if style == triangles:
        triangle( flip ? 좌상 삼각형 : 우하 삼각형 )
```
`strokeCap(SQUARE)`로 선 끝을 각지게 해야 호가 칸 경계에서 매끄럽게 이어진다.

### 주요 파라미터 (Main Parameters)
- `S = 20` : 타일 한 변 픽셀. 작을수록 촘촘한 무늬.
- 각 칸 확률 0.5로 방향 결정.

## 4. 데이터 구조 및 이식 가이드 (Data Structures & Porting Guide)
- 필요한 그리기: 선, 호(arc), 삼각형. 호가 없으면 짧은 선분들로 사분원 근사.
- 확장 아이디어: 방향을 무작위 대신 노이즈/규칙으로 정하면 흐름·미로 제어 가능.
