# 다각형 삼각분할 (Polygon Triangulation) — Ear Clipping

> **[AI Implementation Context]**
> 이 문서는 해당 알고리즘을 다른 프로그래밍 언어(C++, Python, Unity, Rust 등)나 그래픽스 환경으로 이식(Porting)할 때 참조하기 위한 기하학/알고리즘 명세서입니다. 특정 언어의 그래픽스 API에 종속되지 않은 순수 논리와 수학적 규칙을 담고 있습니다.


## 1. 개요 및 목적 (Overview)
단순 다각형(자기교차 없는 폐곡선) 하나를 **삼각형들로 분할**한다. 한 조각씩 애니메이션으로 채워진다.

## 2. 핵심 이론 및 원리 (Core Concept & Math)
다각형의 **귀(ear)** = 볼록한 꼭짓점 세 개(이전-현재-다음)가 이루는 삼각형인데, 그 삼각형 안에 **다른 어떤 꼭짓점도 들어있지 않은** 것. 귀를 찾아 잘라내면 다각형의 꼭짓점이 하나 줄고, 이를 3개 남을 때까지 반복한다.

판정에 쓰는 두 도구:
- **부호 있는 외적** `cross(a,b,c) = (b−a)×(c−a)` : 부호로 좌/우 회전(볼록/오목) 판별.
- **점의 삼각형 내부 판정** : 세 변에 대한 외적 부호가 모두 같은 쪽이면 내부.

## 3. 알고리즘 의사코드 (Pseudocode)
```
idx = 모든 꼭짓점 인덱스 (반시계 방향 가정)
while idx.length > 3:
    for each i:
        a,b,c = 이전, 현재, 다음 꼭짓점
        if cross(a,b,c) <= 0: continue        // 오목(볼록 아님) → 귀 아님
        if 삼각형(a,b,c) 안에 다른 꼭짓점이 있음: continue
        // 귀 발견!
        output.push([a,b,c])
        idx에서 b 제거
        break
남은 3개도 삼각형으로 output에 추가
```

### 주요 파라미터 (Main Parameters)
- 테스트 다각형: 중심 기준 16개 꼭짓점, 반지름 랜덤(75~230)으로 별/톱니 모양.
- 꼭짓점 **감김 방향(winding)** 이 반시계여야 `cross > 0`가 볼록을 뜻한다(시계면 부호 반대).

## 4. 데이터 구조 및 이식 가이드 (Data Structures & Porting Guide)
- 순수 기하 연산(외적)만 필요. 그래픽스 의존 없음.
- 시간복잡도 O(n²). 다각형이 크면 monotone-polygon 분할이나 라이브러리(예: earcut) 사용.
- 볼록 다각형만 필요하면 삼각형 부채꼴(fan) 분할로 충분.
