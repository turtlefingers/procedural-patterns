# 이슬람 기하 패턴 / 기리 (Girih) — Hankin 엮임선 기법

## 무엇을 만드나
정다각형(12각형·6각형·4각형)으로 평면을 빈틈없이 덮은 뒤(**Dodeca–Hexa–Square 타일링**), 각 다각형의 변마다 **Hankin 방법**으로 엮임선(strapwork)을 그어 별·다각형이 얽힌 이슬람 기하 문양을 만든다. 구조는 Daniel Shiffman의 Islamic Pattern Generator를 따른다.

## 핵심 개념
### 1) 타일링 (평면 채우기)
12각형 하나를 중심에 놓으면 그 둘레에 사각형과 육각형이 규칙적으로 들어맞는다. 삼각함수로 각 이웃 다각형의 반지름과 중심 거리를 계산해 셀(cell) 하나를 구성하고, 이 셀을 격자로 반복한다.
- n각형의 외접반지름 r, 변 길이 `side = r·sin(π/n)`, 중심-변 거리(아포템) `h = r·cos(π/n)`
- 인접 다각형의 반지름은 변 길이를 공유하도록 역산: `r_m = side / sin(π/m)`

### 2) Hankin 엮임선 (각 변 → 두 개의 선분)
각 변의 중점에서, 변에 대해 **고정 각도 `angle`** 로 두 방향으로 선을 쏜다. 두 다각형이 변을 공유하므로, 양쪽에서 쏜 선들이 변 위에서 만나 연속된 별무늬가 된다.
```
edge(a, b):
    mid = (a + b) / 2
    v1 = (a - mid) 방향,  v2 = (b - mid) 방향
    // delta 만큼 중점에서 바깥으로 살짝 이동한 두 시작점
    v1 = rotate(v1, -angle);  v2 = rotate(v2, +angle)
    len = sin(α/2) · (half_len + delta) / sin(π - α/2 - angle)   // α = 다각형 내각
    선분1 = (offset1) → (offset1 + v1·len)
    선분2 = (offset2) → (offset2 + v2·len)
```
`α`(내각)와 `angle`(엮임 각도)로 선의 교차점이 정해지고, 이것이 문양의 별 모양을 좌우한다.

## 알고리즘 요약
```
1. buildGrid(): 화면을 덮도록 셀을 격자 배치, 각 셀은 12/6/4각형 여러 개
2. 각 polygon: 꼭짓점 = 중심 + r·(cos θ, sin θ),  θ = i·2π/n - π/n + init_angle
3. 각 polygon의 모든 edge에 hankin(내각) 적용 → 엮임선 2개 생성
4. 렌더: 다각형 채움(색) + 흰 굵은 엮임선 + 얇은 강조선
```

## 주요 파라미터
- `angle`(엮임 각도, 기본 54°): 45/60/75°에 가까우면 선이 중심에 몰려 막히므로 그 근처를 피한다.
- `delta`(중점 오프셋, 5~15): 엮임선 굵기/간격 느낌.
- 다각형 변 개수(12/6/4)에 따라 색을 다르게 칠함.

## 다른 언어로 옮길 때
- 벡터 회전 `rotate(v, θ)`, 벡터 크기 설정 `setMag`만 있으면 된다.
- 핵심 난이도는 **타일링 좌표 계산**(셀 안 다각형들의 상대 위치). 삼각함수 관계식을 그대로 옮기면 된다.
- 엮임선이 변 위에서 정확히 만나려면 인접 다각형이 **변을 공유**해야 한다(타일링이 정확해야 함).
