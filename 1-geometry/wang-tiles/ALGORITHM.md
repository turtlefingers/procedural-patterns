# 왕타일 (Wang Tiles)

> **[AI Implementation Context]**
> 이 문서는 해당 알고리즘을 다른 프로그래밍 언어(C++, Python, Unity, Rust 등)나 그래픽스 환경으로 이식(Porting)할 때 참조하기 위한 기하학/알고리즘 명세서입니다. 특정 언어의 그래픽스 API에 종속되지 않은 순수 논리와 수학적 규칙을 담고 있습니다.


## 1. 개요 및 목적 (Overview)
네 변에 색이 칠해진 정사각 타일들을, **맞닿는 변의 색이 일치**하도록 격자에 배치한다. 각 타일은 중심에서 네 변으로 뻗은 4개의 삼각형(변 색)으로 그린다.

## 2. 핵심 이론 및 원리 (Core Concept & Math)
왕타일의 규칙: 타일을 회전/반사 없이, 인접 변의 색이 같아야만 이어붙일 수 있다. 색 제약을 만족시키며 평면을 채우는 것은 일반적으로 어려운 문제지만(비주기적 타일링·계산 이론과 연결), 여기서는 **왼쪽·위 이웃이 정한 색을 그대로 물려받는** 탐욕적(greedy) 배치로 항상 성공한다.

## 3. 알고리즘 의사코드 (Pseudocode)
```
// tileset[w][n] = 서쪽 변 색 w, 북쪽 변 색 n 을 가진 타일의 (동쪽 e, 남쪽 s) 색
makeTileset(): 모든 (w,n) 조합마다 e,s 를 무작위로 정해 둔다

for cursor = 0 .. GRID*GRID-1:      // 왼쪽→오른쪽, 위→아래
    gx, gy = cursor 위치
    W = (gx>0) ? 왼쪽타일.e : 무작위   // 서쪽 색 = 왼쪽 이웃의 동쪽 색
    N = (gy>0) ? 위쪽타일.s : 무작위   // 북쪽 색 = 위쪽 이웃의 남쪽 색
    t = tileset[W][N]                 // 제약을 만족하는 타일 선택
    이 칸에 (W, N, t.e, t.s) 저장 및 그림
```
왼쪽·위가 이미 색을 고정하므로 자동으로 이음새 색이 일치한다.

### 주요 파라미터 (Main Parameters)
- `NC = 4`(색 개수), `GRID = 10`(격자), `S = 50`(타일 픽셀).
- 타일 렌더: 네 삼각형(북/동/남/서)에 각 변 색을 칠하고 검은 외곽선.

## 4. 데이터 구조 및 이식 가이드 (Data Structures & Porting Guide)
- 삼각형 4개 그리기 + 2D 배열만 있으면 된다.
- 이 방식은 항상 유효한 배치를 보장한다(진짜 왕타일의 미결정성 문제는 회피). 임의의 타일 집합으로 채우려면 역추적(backtracking)/제약 전파가 필요.
