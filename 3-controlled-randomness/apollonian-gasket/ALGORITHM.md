# 아폴로니안 가스켓 (Apollonian Gasket) — 데카르트 원 정리

## 무엇을 만드나
서로 접하는 세 원의 틈에 **그 셋에 동시에 접하는 원**을 채우고, 새로 생긴 틈마다 재귀적으로 반복해 무한히 원이 채워지는 프랙탈.

## 핵심 개념 — 데카르트 원 정리 (Descartes Circle Theorem)
곡률 `k = 1/r`(부호: 안쪽 접함은 +, 바깥을 감싸는 원은 −). 서로 접하는 네 원의 곡률은:
```
k4 = k1 + k2 + k3 ± 2·√(k1·k2 + k2·k3 + k3·k1)
```
± 두 해는 "안쪽 작은 원"과 "바깥 감싸는 원"에 해당. 중심 좌표는 곡률에 위치를 곱한 **복소수** 버전으로 같은 식을 푼다:
```
z4·k4 = z1·k1 + z2·k2 + z3·k3 ± 2·√(z1z2·k1k2 + z2z3·k2k3 + z3z1·k3k1)
```
(복소수 곱·제곱근 필요.)

## 알고리즘 (의사코드)
```
init: 큰 바깥 원(곡률 -1/R) + 그 안에 접하는 두 원 → 삼중항 큐에 넣음
while 큐가 비지 않고 원 수 < 한계:
    (c1,c2,c3) = 큐에서 꺼냄
    k4± = descartes(k1,k2,k3)
    각 k4에 대해 complexDescartes로 후보 원 z4 두 개 계산
    for 후보 c4:
        if validate(c4):            // 반지름 범위 OK & 세 원에 모두 접함 & 중복 아님
            all.push(c4)
            큐에 (c1,c2,c4), (c1,c3,c4), (c2,c3,c4) 추가
그리기: 모든 원의 윤곽선
```
`validate`의 접함 검사: 두 원 거리 d 가 `|r1±r2|` 중 하나와 (오차 EPS 내) 같은가.

## 주요 파라미터
- `MIN_R = 1.1`(더 작아지면 멈춤), `EPS = 0.1`(접함 판정 허용오차).
- 초기 두 내부 원의 반지름 비율을 바꾸면 다른 가스켓.

## 다른 언어로 옮길 때
- **복소수 산술**(add/sub/mul/sqrt)이 필수. 없으면 실수 2벡터로 직접 구현.
- 중복 원 제거(방문 집합/근접 비교)와 접함 허용오차가 안정성의 관건.
- 재귀 대신 큐(BFS)로 폭을 제어하면 애니메이션·성능에 유리.
